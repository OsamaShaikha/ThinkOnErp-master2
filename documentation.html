📋Project Overview

ThinkOnErp is a comprehensive Enterprise Resource Planning (ERP) API designed for multi-tenant organizations. It provides secure role-based management of companies, branches, users, currencies, fiscal years, permissions, and a full support ticket system. On top of the core ERP, a massive Full Traceability System provides audit logging, compliance reporting, performance monitoring, security threat detection, and alerting.

🏗️Architecture

The system follows Clean Architecture with strict dependency rules. All dependencies point inward — the Domain layer has zero external dependencies.

🌐 API Layer — Controllers, Middleware, Program.cs
▼
⚙️ Application Layer — MediatR Commands/Queries, DTOs, Validators, Behaviors
▼
💎 Domain Layer — Entities, Interfaces, Models, Exceptions (zero dependencies)
▲
🔧 Infrastructure Layer — ADO.NET Repositories, JWT Service, Audit Services, Oracle DB

Request Pipeline Flow

HTTP Request→ RequestTracingMiddleware→ ExceptionHandlingMiddleware→ JWT Authentication→ ForceLogoutMiddleware→ Authorization→ Controller→ MediatR Pipeline→ Handler→ Repository → Oracle

🔧Infrastructure Layer

Contains 23 repositories, 45+ services, resilience patterns, and 27 configuration option classes. All database operations use stored procedures via ADO.NET.

High-performance batch audit writer using System.Threading.Channels. Default: 50 events/batch, 100ms window.

Wraps AuditLogger with circuit breaker pattern for database failure resilience.

SecurityMonitor

Real-time threat detection: SQL injection, XSS, brute force, anomalous behavior.

ComplianceReporter

AlertManager

AuditLogIntegrityService

KeyManagementService

Encryption/signing key lifecycle with automated rotation.

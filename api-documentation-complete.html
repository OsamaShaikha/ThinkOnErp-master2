📖 Overview

ThinkOnERP provides a comprehensive REST API with 21 controllers covering authentication, business operations, monitoring, compliance, and system management.

Base URLs

  • HTTPS: https://localhost:7136
  • HTTP: http://localhost:5160
  • Swagger UI: https://localhost:7136/swagger

Authentication

Most endpoints require JWT authentication. Include the token in the Authorization header:

Authorization: Bearer eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...

Response Format

All API responses follow a consistent format:

{
  "success": true,
  "statusCode": 200,
  "message": "Operation completed successfully",
  "data": { ... },
  "errors": null,
  "timestamp": "2026-05-06T00:00:00Z",
  "traceId": "abc123..."
}

1. 🔐 Authentication API

Base Route: /api/auth

Purpose: User authentication, token management, and session control

Key Features

  • JWT-based authentication
  • Refresh token support
  • Dual authentication (users and super admins)
  • SHA-256 password hashing
  • Public endpoints (no authentication required)

Endpoints

POST /api/auth/login Public

Purpose: Authenticate a regular user and generate JWT tokens

Request Body:

{
  "userName": "moe",
  "password": "Admin@123"
}

Success Response (200 OK):

{
  "success": true,
  "data": {
    "token": "eyJhbGciOiJIUzI1NiIs...",
    "refreshToken": "a1b2c3d4-e5f6-7890...",
    "expiresAt": "2026-05-07T00:00:00Z",
    "refreshTokenExpiresAt": "2026-05-13T00:00:00Z",
    "userId": 5,
    "userName": "moe",
    "userType": "Admin"
  }
}
Use Cases:
  • User login to access the system
  • Mobile app authentication
  • Web application login
POST /api/auth/refresh Public

Purpose: Get a new access token using a valid refresh token

Request Body:

{
  "refreshToken": "a1b2c3d4-e5f6-7890-abcd-ef1234567890"
}
Use Cases:
  • Refresh expired access token without re-login
  • Maintain user session seamlessly
  • Background token refresh in mobile apps
POST /api/auth/superadmin/login Public

Purpose: Authenticate a super administrator

Request Body:

{
  "userName": "superadmin",
  "password": "Admin@123"
}
POST /api/auth/superadmin/refresh Public

Purpose: Refresh super admin access token

💡 Security Note: All passwords are hashed with SHA-256 before storage. JWT tokens expire after 60 minutes by default, while refresh tokens last 7 days.

📚 Best Practices

1. Authentication

  • Always use HTTPS in production
  • Store tokens securely (not in localStorage)
  • Implement automatic token refresh
  • Handle 401 errors gracefully

2. Error Handling

  • Check the success field in responses
  • Display user-friendly error messages
  • Log errors with traceId for debugging
  • Implement retry logic with exponential backoff

3. Performance

  • Use pagination for large datasets
  • Implement caching where appropriate
  • Monitor slow requests and queries
  • Use filtering to reduce data transfer

4. Security

  • Validate all input on client side
  • Never expose sensitive data in URLs
  • Implement rate limiting on client side
  • Use CORS properly in production

5. Monitoring

  • Monitor API response times
  • Track error rates and patterns
  • Set up alerts for critical issues
  • Review audit logs regularly

🔄 Common Workflows

1. User Registration and Login

Steps:
  1. Admin logs in: POST /api/auth/login
  2. Admin creates user: POST /api/users
  3. User logs in: POST /api/auth/login
  4. User changes password: PUT /api/users/{id}/change-password

2. Create Company with Branch

Steps:
  1. Super admin logs in: POST /api/auth/superadmin/login
  2. Create company with default branch: POST /api/companies
  3. Create additional branches: POST /api/branches
  4. Create users for branches: POST /api/users

3. Support Ticket Lifecycle

Steps:
  1. User creates ticket: POST /api/tickets
  2. Admin assigns ticket: PUT /api/tickets/{id}/assign
  3. Support adds comments: POST /api/tickets/{id}/comments
  4. Update status: PUT /api/tickets/{id}/status
  5. Close ticket: PUT /api/tickets/{id}/status (status: Closed)

4. Generate Compliance Report

Steps:
  1. Admin logs in: POST /api/auth/login
  2. Generate GDPR report: GET /api/compliance/gdpr/data-export
  3. Export to CSV: Add ?format=csv parameter
  4. Download and provide to data subject

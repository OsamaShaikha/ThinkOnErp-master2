# User Guide: Audit Log Querying

## Overview

The ThinkOnErp audit system records every data change, authentication event, permission modification, and system exception. This guide shows how to query and analyze audit data.

---

## Authentication

All audit endpoints require admin-level JWT authentication:

```http
POST /api/auth/login
Content-Type: application/json

{ "userName": "admin", "password": "your-password" }
```

Use the returned `accessToken` in all subsequent requests:
```
Authorization: Bearer {accessToken}
```

---

## Querying Audit Logs

### Basic Query

```http
GET /api/auditlogs?page=1&pageSize=20
Authorization: Bearer {token}
```

### Filter by Date Range

```http
GET /api/auditlogs?startDate=2024-01-01&endDate=2024-01-31&page=1&pageSize=50
```

### Filter by Entity Type

```http
GET /api/auditlogs?entityType=SysUser&page=1&pageSize=20
```

### Filter by Action

```http
GET /api/auditlogs?action=UPDATE&page=1&pageSize=20
```

Actions: `INSERT`, `UPDATE`, `DELETE`, `LOGIN`, `LOGOUT`, `FAILED_LOGIN`, `PERMISSION_CHANGE`

### Filter by User (Actor)

```http
GET /api/auditlogs?actorId=123&page=1&pageSize=20
```

### Filter by Company and Branch

```http
GET /api/auditlogs?companyId=1&branchId=5&page=1&pageSize=20
```

### Filter by Severity

```http
GET /api/auditlogs?severity=ERROR&page=1&pageSize=20
```

Severities: `INFO`, `WARNING`, `ERROR`, `CRITICAL`

### Filter by Business Module

```http
GET /api/auditlogs?businessModule=Accounting&page=1&pageSize=20
```

Modules: `POS`, `HR`, `Accounting`, `Finance`, `Administration`, `Security`

### Combine Multiple Filters

```http
GET /api/auditlogs?companyId=1&entityType=SysUser&action=UPDATE&startDate=2024-01-01&endDate=2024-01-31&page=1&pageSize=20
```

---

## Full-Text Search

```http
GET /api/auditlogs/search?query=password+changed&page=1&pageSize=20
```

Searches across: entity type, action, business description, error code, and metadata fields.

---

## View Entity History

See the complete change history for a specific entity:

```http
GET /api/auditlogs/entity-history?entityType=SysUser&entityId=42
```

Returns chronological list of all changes with before/after values.

---

## Trace Request by Correlation ID

Follow a complete request through the system:

```http
GET /api/auditlogs/correlation/{correlationId}
```

Shows all audit entries generated by a single API request, including:
- Middleware request capture
- MediatR command execution
- Database operations
- Any errors encountered

---

## Legacy Dashboard View

For the traditional status-based dashboard:

```http
GET /api/auditlogs/legacy-dashboard
```

Returns counters: Unresolved, In Progress, Resolved, Critical

```http
GET /api/auditlogs/legacy-view?status=Unresolved&page=1&pageSize=20
```

### Update Error Status

```http
PUT /api/auditlogs/legacy/{id}/status
Content-Type: application/json

{
  "status": "Resolved",
  "resolutionNotes": "Fixed null reference in user service",
  "assignedToUserId": 5
}
```

---

## Export Audit Data

```http
GET /api/auditlogs/export?startDate=2024-01-01&endDate=2024-01-31&format=csv
```

Formats: `json`, `csv`, `pdf`
